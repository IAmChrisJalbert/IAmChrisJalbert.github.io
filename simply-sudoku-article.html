Technical Deep Dive

Algorithms of Logic: Sudoku

How recursive backtracking enables Simply Sudoku to generate unique, solvable puzzles in milliseconds.

Sudoku is more than a game—it's a perfect playground for computer science. When I set out to build **Simply Sudoku**, I wanted to ensure that every puzzle was generated on-device, 100% unique, and guaranteed to be solvable with pure logic.

The Engine: Backtracking Algorithm

At the heart of the app is a recursive backtracking algorithm. This approach works by attempting to place a number in a cell and then recursively trying to fill the rest of the board. If it hits a dead end (where no valid number can be placed), it "backtracks" and changes the previous number.

Generating the Challenge

Generating a board is a two-step process. First, we generate a "Perfect Board"—a fully solved Sudoku grid. Then, we start removing numbers. The difficulty setting (Easy, Medium, Hard) determines how many "holes" are punched into the grid.

To keep the gameplay fair, the engine runs a solve check after every number is removed to ensure that the board still has one, and only one, unique solution.

Performance on Mobile

While recursive functions can be memory-heavy, Kotlin's efficient handle of stack operations allows Simply Sudoku to solve or generate a board in less than 20ms on most modern Android devices. This ensures that when a user taps "New Game," the board is ready before the screen transition finishes.

"Code should be as elegant as the logic it seeks to solve. Simply Sudoku is a celebration of that principle."
